S2 Sea-Ice Segmentation — Reproduction vs. Paper

Report date: 2026-06-15 (Run A + Run B comprehensive vs paper; prior updates 2026-06-13, 06-11, 06-08)  ·  Workflow: s2-segmentation-workflow (Pegasus, auto-label U-Net)
Paper: Iqrah, Wang, Xie, Prasad — "A Parallel Workflow for Polar Sea-Ice Classification using Auto-labeling of Sentinel-2 Imagery," IEEE IPDPSW 2024.
Reference code: S2_Parallel_Workflow/ (s2_u_net_tf.py, s2_u_net_horovod.py, two notebooks).

1. Executive summary

We reproduced the paper's U-Net-Auto pipeline (HSV color-segmentation auto-labeling + U-Net training) as a Pegasus workflow and trained models on both unfiltered and thin-cloud/shadow-filtered Sentinel-2 inputs. Two full distributed runs anchor the comparison: Run A (scene-scale filter, the paper's config) and Run B (per-tile filter). §1A is the comprehensive, claim-by-claim comparison of both runs against the paper, including a table of what could not be compared and why.

Headline finding (pegasus2-run0002, 2026-06-13 — current canonical run). The paper-default configuration (now the generator's defaults: scenes resized in-DAG to 2048×2048, 256×256 tiles, auto-label, both branches, scene-scale filter, stratified eval, whole-scene inference) reaches 96.25% accuracy on original imagery (paper 90.18%, +6.07 pt) and 99.76% on thin-cloud/shadow filtered (paper 98.97%, +0.79 pt). Resizing to 2048 eliminates the padding-class artifact that capped pegasus2-run0001's orig branch at 91.21% — the orig branch is now a clean 3-class problem and thin-ice recall recovers from 0.63 to 0.89. Cloud-stratified (Table V): orig high-cloud 94.32% / low-cloud 97.67%; filtered 99.64% / 99.86%. See §8.
Headline finding (run0009). With self-consistent filtered labels (--filtered-labels filtered, now the default), the filtered U-Net reaches 99.82% accuracy — slightly exceeding the paper's reported 98.97%. The unfiltered branch reaches 94.72%, also above the paper's 90.18%. The earlier run0008 discrepancy (filtered = 90.86%) was caused by feeding the U-Net filtered inputs while keeping labels from the raw scene; regenerating labels from the filtered images closes the gap and confirms the hypothesis advanced in our prior report.
Update (pegasus2-run0001, 2026-06-11). The first full end-to-end distributed run (14,317 jobs across the MAX/NCSA/HAWI GPU pool: parallel color-seg → both training branches → stratified evaluation → whole-scene inference on all 63 scenes × 2 models) completed with 0 failed jobs after two fixes (missing filter_image.py staging for infer jobs; evaluate-job memory 4→8 GB). Headline accuracy: unfiltered 91.21% (paper 90.18%), filtered 97.71% (paper 98.97%). Numbers are lower than run0009 because this run used the native 2000×2000 scenes zero-padded to tile evenly (run0009 used 2048×2048 resamples): the orig branch's labels are split from the full-scene mask, so the padding becomes a spurious 4th "padding" class (2.54 M test pixels, recall 0.25) that drags down the headline number. The filtered branch re-derives labels from the already-padded tiles, so it has no padding class. See §8.

1A.6 Interpreting Run A vs Run B — variance vs. filter scale

  • Original branch = variance baseline. A and B feed the orig branch identical tiles, so its gaps are pure unseeded training variance: overall accuracy 96.25 vs 94.02 (2.2 pt) and thin-ice recall swings ~12 pt (Run A <10% 84.8% vs Run B 72.8%). Treat any single-run gap of this size as noise.
  • Filtered branch = the real filter-scale comparison. Here the gap is larger than the variance baseline and concentrated in thin ice: Run A (scene filter) holds thin-ice recall at 99.8% overall, while Run B (per-tile filter) drops to 81.1% overall and 60.5% on the clearest (<10% cloud) tiles. The per-tile medianBlur(19), lacking scene context, distorts thin-ice tiles enough that the U-Net reads them as thick. This 39-pt low-cloud gap exceeds the ~12-pt orig variance, so it is most likely a genuine penalty of per-tile filtering — though seeded repeats are needed for an error bar.
  • The zero-cloud bug interacted with this. Run B's in-DAG stratified eval dropped the 127 zero-cloud tiles — exactly the clear, thin-ice-heavy tiles its filter handles worst — so its raw per-stratum PNGs (thin recall ~98%) flattered it. The §1A.4 matrices use the corrected full sets.

7. Conclusions

  1. The pipeline is a faithful reproduction of the paper's U-Net-Auto pipeline. With Option A enabled by default, both the unfiltered (94.72%) and filtered (99.82%) conditions match or exceed the paper's reported numbers (90.18% / 98.97%).
  2. The paper's filtered-condition methodology is now fully pinned down: labels must be regenerated from the filtered images. The paper does not state this explicitly; the most natural reading (filter inputs only) yields ~90% — we suggest the paper clarify this.
  3. The +8.96 pt improvement we observe when switching to self-consistent labels closely tracks the paper's +8.79 pt original→filtered improvement, providing strong evidence that this is the methodology used to produce Table IV.
  4. The Pegasus workflow now offers both conditions as a single DAG via --paths both --filtered-labels filtered, giving a controlled comparison from a single submission.

8. Full distributed runs — pegasus2-run0001 & run0002

Two production-scale executions of the complete DAG (color-seg auto-labeling → both training branches → evaluation → cloud-stratified evaluation → whole-scene inference) on a multi-site HTCondor GPU pool (MAX, NCSA, HAWI workers; 14,317 jobs each). run0001 (2026-06-11) used native 2000×2000 scenes zero-padded to 256-divisible — this exposed the padding-class artifact below. run0002 (2026-06-13, current canonical) resizes every scene to 2048×2048 first, so tiles divide evenly and no padding enters the labels; it also pads mask edges with the water value as a belt-and-suspenders guard. §8.1–8.2 analyse run0001's artifact; §8.4 reports run0002's results.

8.1 The padding-class artifact (run0001 — resolved in run0002)

The orig branch's labels are produced by color-segmenting the full scene and then splitting the mask with --pad; the zero padding survives as a distinct gray value and the LabelEncoder auto-detects 4 classes instead of 3:

The padding class is ~4.8% of test pixels with recall 0.25 — it alone accounts for most of the gap between this run's 91.21% and run0009's 94.72% on the orig branch. The filtered branch re-derives labels by color-segmenting the already-padded tiles, so padding is absorbed into the water class and only 3 classes exist (thin 0.903 / thick 0.983 / water 1.000 F1).

8.3 Whole-scene inference

126 full-scene segmentations (63 scenes × {orig, filtered} models) staged out to output/ as {orig,filtered}_infer_s2_vis_NN.png. Two workflow bugs were found and fixed during this run: (1) infer_unet imports filter_image at module level but the generator did not stage filter_image.py — all 126 infer jobs failed until it was added to the job inputs; (2) evaluate_model exceeded its 4 GB cgroup limit and was bumped to 8 GB. Both fixes are now in workflow_generator.py.

Known defect in this run's rendered artifacts: a post-run code audit (2026-06-11) found infer_unet.py had the thick/thin colour mapping inverted (gray 29 is blue/thin and 76 is red/thick, not vice versa) — the 126 inference PNGs from this run render thick ice blue and thin ice red. Plot axis labels (confusion matrices, metrics tables) carried the same swapped names. The numeric metrics are unaffected (training and evaluation use encoded integer labels). Fixed in bin/infer_unet.py, bin/evaluate_stratified.py, bin/generate_plots.py, and compare_with_paper.py; run0002 produces correctly-coloured/labelled artifacts.

9. Recommended next steps

  • Seeded / repeated runs — training is unseeded beyond the split (matching the reference); repeat the paper-default run a few times to quantify the ~1 pt run-to-run variance and report a mean ± spread for publication.
  • 66-scene parity — recover s2_vis_56/57/64 (or document why they are missing) so the reproduction matches the paper's stated dataset size exactly (we use the 63 the reference code's train_images_4032/ implies).
  • Per-tile filter parity — run the --filter-scale tile variant (Run B) and check whether the per-class numbers change measurably vs the scene-scale default.
